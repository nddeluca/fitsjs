Read a data unit and initialize an appropriate instance depending on the type of data unit (e.g. image, binary table, ascii table). Note: Bytes are not interpreted by this function. That is left to the user to call when the data is needed.

readData: (header) -> return unless header.hasDataUnit() if header.isPrimary() data = new Image(@view, header) else if header.isExtension() if header.extensionType is "BINTABLE" if header.contains("ZIMAGE") data = new CompImage(@view, header) else data = new BinaryTable(@view, header) else if header.extensionType is "TABLE" data = new Table(@view, header) else if header.extensionType is "IMAGE" data = new Image(@view, header) excess = File.excessBytes(data.length)
